Certificate & SSO Integration – ADFS with PHP Web App

🛠️ Project: Integration of ADFS with a PHP Web Application on IIS
As part of my advanced learning in infrastructure and security, I completed a full implementation of Active Directory Federation Services (ADFS) to enable Single Sign-On (SSO) for a PHP web application hosted on a non-domain server.

🎯 Objective

Allow a domain user to authenticate via ADFS on an external web application, without the app being joined to the Active Directory domain.

🔧 Environment

  • ADDS, ADFS, ADCS (PKI) on multiple servers across three simulated sites (Winnipeg, Toronto, Ottawa)
  • Non-domain web server (SVRWEB) on IIS
  • Inter-site routing and custom DNS configuration
  • Simple PHP app (with WS-Federation redirection) on IIS

⚙️ Key Steps

  • Setup of a root and subordinate certification authority
  • Enabled secure LDAPS protocol for Active Directory
  • Configured ADFS with KDS key and SSL certificate signed by internal CA
  • Established trust relationship between ADFS and the web app
  • Installed PHP 8.2 with FastCGI on IIS
  • HTTPS link via manually generated certificate deployed through /certsrv
  • Implemented ADFS redirection in the PHP app
  • Full test from a non-domain client workstation

✅ Result

A domain user can access the web app hosted on SVRWEB via:
https://webapp.alag.local/
and authenticate securely through ADFS, with automatic redirection and custom post-login display.

🧠 Technical Challenges Overcome

  • IIS configuration errors (500.19), certificate issues (unsecured)
  • DNS resolution between domain and external machines
  • Manual certificate requests via /certsrv web interface
  • Fine-tuned NTFS permissions and required IIS modules (FastCGI, URL Rewrite)

📌 Skills Developed

  • ADFS integration and web access security
  • Windows Server administration (AD, ADFS, IIS, PKI)
  • Certificate and GPO management
